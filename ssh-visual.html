Passwordless SSH
— how it actually works

Stop typing passwords. Here's the key pair model explained from scratch.

arjun@macbook ~ $
1
Generate Key Pair
$ ssh-keygen -t ed25519 -C "arjun@devops" # modern algo
Creates two files: your private key (keep secret) and public key (safe to share). ed25519 is the modern standard — RSA is older and slower.
2
Copy Public Key to Server
$ ssh-copy-id user@server-ip # one-time setup
Pushes your public key into the server's ~/.ssh/authorized_keys. After this — no more password prompts.
3
Connect — Zero Password
$ ssh prod-bastion # instant access
SSH checks your private key against the authorized public key on the server. Match = access. Using an alias from ~/.ssh/config — no IP needed.

// the lock & key model

Public key = the lock — lives on the server, safe to share.
Private key = your key — stays on your machine, never shared.
SSH checks if your key opens the lock. If yes — you're in.

~/.ssh/authorized_keys
Public keys of clients allowed to connect
~/.ssh/known_hosts
Servers your machine has verified and trusts
~/.ssh/config
Aliases for prod, staging, bastion, db — switch with ssh <alias>
