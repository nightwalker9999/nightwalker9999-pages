dns-resolution

I thought I understood DNS.
I didn't.

What I assumed

When the resolver can't find something in cache, it asks a well-known server like Cloudflare or Google DNS — and that server knows the IP.

What actually happens

DNS is decentralised. The resolver doesn't ask someone who knows. It asks someone who knows who to ask next.

Root server → .com TLD → Authoritative NS → IP address

The cache can short-circuit this at any stage. The tree only gets climbed as far as necessary.

I only found this gap when someone asked me to explain it out loud. I stumbled mid-sentence.

That stumble was worth more than re-reading the docs three times.
